FAQs
How is my Fuel Score calculated? ▼
Your Fuel Score (0–100) combines calorie adherence (40%), protein target (25%), water intake (20%), and activity (15%). Hit all four and you'll earn a lime dot.
Can I change my calorie target? ▼
Yes. Go to Profile → Edit Profile → Daily Calorie Target and update it anytime. Your dot history stays intact.
How does the AI Meal Logger work? ▼
Type any meal name (e.g. "Dal rice" or "Chicken biryani 1 plate") in the ✨ AI Log section. Our AI instantly estimates calories, protein, carbs and fat. Tap LOG to save it.
Why isn't my Strava syncing? ▼
Go to Profile and tap the ↻ Sync button on the Strava card. If it says "Token Refreshed", tap Sync once more. Strava access tokens expire every 6 hours and are refreshed automatically.
What is a Daily Challenge? ▼
Every day a new AI-generated fitness challenge appears in the Challenges tab. Complete it and tap Done to count it towards your daily streak. Challenges reset at midnight.
What is a Rescue Token? ▼
Each challenge gives you one Rescue Token. Use it to cover a missed day without breaking your streak. Each token can only be used once per challenge.
How do I delete my account? ▼
Email dotcreations404@gmail.com and we'll delete your account and all data within 7 days, in line with our privacy policy.

What is DotFuel?
DotFuel is a minimalist nutrition tracker built around one idea: your entire day summarised as a single coloured dot. Log meals, track macros, stay hydrated, and complete challenges. Every day, one dot tells the story.
The Dot System

Green: Good day, close to target (65–84)
Blue: Under-fuelled (40–64)
Red: Over budget (<40)
Gray: No data logged

Database Setup
The challenge tables don't exist yet.
Copy the SQL below and run it in your Supabase Dashboard → SQL Editor.
Step 1 — Open SQL Editor
Go to supabase.com/dashboard → select your project → click SQL Editor in the left sidebar.
Step 2 — Paste & Run this SQL
-- DOT CHALLENGE Vol3 — Full Setup
-- Run this in your Supabase SQL Editor

-- 1. Challenge Participants Table
create table if not exists public.challenge_vol3_participants (
    id uuid default gen_random_uuid() primary key,
    user_id uuid references auth.users(id) on delete cascade not null,
    joined_at timestamp with time zone default timezone('utc'::text, now()) not null,
    cut_list text[] default '{}'::text[] not null,
    custom_task_title text default 'My custom daily habit',
    revivals_remaining integer default 2 not null,
    status varchar default 'active' check (status in ('active', 'failed', 'completed')) not null,
    unique(user_id)
);

-- 2. Daily Progress Tracking
create table if not exists public.challenge_vol3_daily_progress (
    id uuid default gen_random_uuid() primary key,
    user_id uuid references auth.users(id) on delete cascade not null,
    log_date date default current_date not null,
    clean_meals boolean default false not null,
    workout boolean default false not null,
    read_page boolean default false not null,
    custom_task_done boolean default false not null,
    water_synced_override boolean default false not null,
    is_calculated_success boolean default false not null,
    revival_applied boolean default false not null,
    unique(user_id, log_date)
);

-- 3. Secure Chat Table
create table if not exists public.challenge_vol3_chat (
    id uuid default gen_random_uuid() primary key,
    user_id uuid references auth.users(id) on delete cascade not null,
    profile_name text,
    message text,
    image_url text,
    created_at timestamp with time zone default timezone('utc'::text, now()) not null
);

-- 4. Enable RLS
alter table public.challenge_vol3_participants enable row level security;
alter table public.challenge_vol3_daily_progress enable row level security;
alter table public.challenge_vol3_chat enable row level security;

-- 4. RLS Policies
create policy "Users can view and manage their own participation"
    on public.challenge_vol3_participants for all using (auth.uid() = user_id);

create policy "Allow authenticated users to read participants"
    on public.challenge_vol3_participants for select
    to authenticated
    using (true);

create policy "Active participants can read the feed"
    on public.challenge_vol3_chat for select
    using (exists (select 1 from public.challenge_vol3_participants where user_id = auth.uid() and status = 'active'));

create policy "Active participants can post to the feed"
    on public.challenge_vol3_chat for insert
    with check (auth.uid() = user_id and exists (select 1 from public.challenge_vol3_participants where user_id = auth.uid() and status = 'active'));

-- 5. Enable Realtime for Chat Feed
alter publication supabase_realtime add table public.challenge_vol3_chat;
Step 3 — Reload the App
After running the SQL, come back here and pull down to refresh or reload the page. The challenge will load automatically.
